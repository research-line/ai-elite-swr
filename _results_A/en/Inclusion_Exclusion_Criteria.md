## 2. Inclusion/Exclusion Criteria (Phase 5.02-5.03)

### 2.1 Methodological Foundation

The research project follows **Grounded Theory** approach (Glaser/Strauss).
Core principle: **When in doubt, include.** Data that may be relevant must not be prematurely excluded. Theoretical saturation is determined by analysis itself, not pre-filtering.

### 2.2 Inclusion Criteria (inclusion = 1)

A statement is **included** if it meets AT LEAST ONE of the following criteria:

| Nr. | Criterion | Operationalization |
|---|---|---|
| E1 | **Worldview Relevance** | Statement contains at least 1 primary category from coding system (FO, SB, AR, WV, EP, RI, ET, GE, MA, RE, MB, SP/EX, TR) |
| E2 | **Thematic Reference** | Statement clearly refers to AI, technology, society, progress, ethics, human nature or related fields |
| E3 | **Action Reference** | Statement reveals attitude, intention or action orientation regarding technological development |
| E4 | **Biographical Relevance** | Statement is biographical but directly suggests conclusion about worldview/value system |

### 2.3 Exclusion Criteria (inclusion = 0)

A statement is **excluded** if it meets ALL of the following conditions simultaneously:

| Nr. | Criterion | Operationalization |
|---|---|---|
| A1 | **No Primary Category** | Statement has zero primary categories in coding system |
| A2 | **No Worldview Substance** | Statement contains no recognizable position on AI, technology, society, values or future |
| A3 | **Pure Anecdotal/Trivial** | Statement is pure anecdote, everyday statement or organizational memo without substantive content |

### 2.4 Special Cases: Duplicates

| Type | Treatment |
|---|---|
| **Exact duplicate, same person** | Keep earlier version; exclude later version (inclusion=0) |
| **Exact duplicate, different persons** | Keep all (inclusion=1) — each person analyzed individually |
| **Near-duplicate, same person** | Keep more complete version; shorter version conditionally excluded |
| **Near-duplicate, different persons** | Keep all (inclusion=1) |

### 2.5 Edge Case Marking (edge_case = 1)

Statements included but marked as **edge cases** for later sensitivity analysis:

| Nr. | Criterion | Operationalization |
|---|---|---|
| G1 | **Very Short, Context-Poor** | Statement under 30 characters AND without clear thematic reference |
| G2 | **Secondary Codes Only** | Statement has primary codes, but assignment is ambiguous |
| G3 | **Questionable Attribution** | Unclear whether statement really belongs to person |

---

## 3. Conservatism Principle

**Core principle**: Excluding a potentially relevant statement is a greater error than including an irrelevant one.

In practice, this means:
- Short statements like "Competition is for losers" or "AI is a new digital species" are INCLUDED despite brevity — they contain clear worldview positions
- Undated statements are INCLUDED, as missing date tells nothing about substantive relevance
- CAIS Statement (4x identical) is INCLUDED for EACH person, as it documents individual positioning

### 4.4 Not Excluded (Justification)

The following statement types were deliberately NOT excluded:

- **71 very short statements** (under 50 chars): Contain clear worldview positions despite brevity (e.g., "Competition is for losers", "Feel the AGI!")
- **82 undated statements**: Missing date is not a substantive criterion
- **8 duplicates of different persons** (CAIS Statement 4x, "Future is bright" 2x, "What models can do" 2x): Document individual positioning of each person
- **3 statements without primary category**: All 3 have recognizable thematic reference and were included as edge cases

### 4.5 Methodological Reflection

The extremely low exclusion rate (0.17%) confirms:

1. **Quality of data collection**: Statements were already carefully curated in previous phases for thematic relevance
2. **Density of coding**: With average 3.5 codes per statement and only 1 statement with single code, coding system is consistently applied
3. **Conservative principle works**: No substantively relevant data point was lost. The 3 exclusions concern only redundant shortened versions of already completely captured statements
