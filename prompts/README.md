# SWR Prompt Templates

This directory contains the prompt templates used in the Synthetic Worldview Reconstruction (SWR) method.

## Prompt Overview

| File | Operationalization | Purpose |
|------|-------------------|---------|
| `op1_core_synthesis.md` | Op1 (Core Synthesis) | Create fictional synthetic person from group data |
| `op2_rating.md` | Op2 (Dimensional Rating) | Rate synthesis on D01-D12 dimensions |
| `op4_crossmodal_prediction.md` | Op4 (Cross-Modal Prediction) | Predict actions from statements (or vice versa) |
| `op6_comparison.md` | Op6 (Structured Comparison) | Compare two group worldviews in dialogue |
| `distillate.md` | Distillation | Compress worldview to 5 core sentences |

## Usage

Each prompt expects a `[SYNTHESIS UNIT]` or `[SYNTHESIS]` placeholder to be replaced with the actual data. Synthesis units are generated by `_data/generate_topf.py` from the database.

## Language

Prompts are in English. The method can be applied in any language by translating the prompt templates. The core logic (inverse problem, fictional person, dimensional rating) is language-independent.

## Models Used in the Pilot Study

- **Claude Sonnet 4.5** (`claude-sonnet-4-5-20251001`): All D01-D12 ratings (Op2), 9 parallel batch agents
- **Claude Opus 4.6** (`claude-opus-4-6`): Core syntheses (Op1), comparisons (Op6), cross-modal prediction (Op4), all validation experiments
- Temperature: 0 for all runs
- No other LLM was used

## Individual-Level Application

The same prompts can be used for individual-level analysis (single person's data as input). However, individual-level application is methodologically more demanding because:
- Blinding becomes critical (the LLM may recognize the person)
- The contamination risk is higher (training data may contain biographical knowledge)
- The distinction between reconstruction and reproduction is harder to establish

For details, see Paper B (SWR method paper), Section 3.1.
